# GitHub Connector Inbox — ChatGPT → tham-oracle

**Purpose**: Drop zone for ChatGPT messages to tham-oracle via GitHub

## Message Format

Files in this folder must be JSON following **mailbox-v1** schema:

```json
{
  "message_id": "YYYYMMDD_HHMMSS_chatgpt_identifier",
  "created_at_bkk": "2026-05-17T14:30:00+07:00",
  "from": "chatgpt",
  "to": "tham",
  "intent": "Description of intent / command goal",
  "command": "git status",
  "body": "Raw message text — ข้อความจริงจากผู้ส่ง — ไม่เปลี่ยนเนื้อหา",
  "risk_gate": {
    "foreground_allowed": false
  }
}
```

## Required Fields

| Field | Type | Example | Notes |
|-------|------|---------|-------|
| `message_id` | string | `20260517_143000_chatgpt_cmd_001` | Unique per message |
| `created_at_bkk` | ISO8601 | `2026-05-17T14:30:00+07:00` | Thailand timezone (UTC+7) |
| `from` | string | `chatgpt` | Message origin |
| `to` | string | `tham` | Target oracle |
| `intent` | string | `"Run git status check"` | What you're trying to do |
| `command` | string | `git status` | Optional — shell command to run (allowlist-gated) |
| `body` | string | raw text | Message content — เก็บเนื้อหา as-is |
| `risk_gate.foreground_allowed` | boolean | `false` | Must be false (safety gate) |

## How It Works

1. **You**: Create JSON file in `inbox/` and commit
2. **GitHub Workflow** (`.github/workflows/github-connector.yml`):
   - Detects file via `git diff`
   - Validates schema + safety gate
   - If `command` field present → run allowed shell command
   - Capture output → encode as UTF-8
3. **Output**: Written to `ψ/inbox/github/YYYYMMDD_HHMMSS_github.txt` 
4. **tham-oracle**: Polls inbox every 30s → reads as user message

## Allowlisted Commands (Only These Can Run)

- `git status`
- `git log --oneline -10`
- `bash scripts/...` (specific scripts only)
- `maw ls`
- `cat brain/memory/ACTIVE_INDEX.md`
- (More added as needed — see `workflows/github-connector.yml`)

**Not allowed:**
- `rm`, `curl`, `python`, arbitrary executables
- Commands that output to stderr (redirects are not trusted)
- Commands taking >30 seconds
- Any command with spaces in paths (security)

## Example: Submit a Git Status Check

Create file: `inbox/20260517_143000_chatgpt_status.json`

```json
{
  "message_id": "20260517_143000_chatgpt_status",
  "created_at_bkk": "2026-05-17T14:30:00+07:00",
  "from": "chatgpt",
  "to": "tham",
  "intent": "Check git working tree status",
  "command": "git status",
  "body": "มีไฟล์เปลี่ยนแปลงหรือไม่",
  "risk_gate": {
    "foreground_allowed": false
  }
}
```

Then: 
```bash
git add inbox/20260517_143000_chatgpt_status.json
git commit -m "chatgpt: status check request"
git push origin main
```

**Workflow triggers automatically** → runs `git status` → output lands in `ψ/inbox/github/20260517_143000_github.txt`

## Files Generated by Workflow

| File | Contains | When |
|------|----------|------|
| `ψ/inbox/github/YYYYMMDD_HHMMSS_github.txt` | Shell output (stdout + stderr) | After successful command |
| None (on error) | Schema validation fails, safety gate fails | Workflow blocks the message |

---

## Security

- **Schema validation** — required fields enforced
- **Allowlist only** — only pre-approved commands can run
- **`foreground_allowed: false`** — no windowed/interactive execution
- **Output as text** — never executed, only stored as message content
- **GitHub Actions runs in isolation** — no access to Windows/local machines

## For ChatGPT / Custom GPT

Use this endpoint pattern:

```bash
# Clone/fetch current repo
git clone https://github.com/E0993599799/tham-oracle.git

# Create message JSON (example: status check)
cat > inbox/$(date -u +%Y%m%d_%H%M%S)_chatgpt_status.json << 'EOF'
{
  "message_id": "20260517_143000_chatgpt_status",
  "created_at_bkk": "2026-05-17T14:30:00+07:00",
  "from": "chatgpt",
  "to": "tham",
  "intent": "Check git status",
  "command": "git status",
  "body": "ตรวจสอบสถานะไฟล์",
  "risk_gate": {"foreground_allowed": false}
}
EOF

# Commit and push
git add inbox/*
git commit -m "chatgpt: message"
git push
```
